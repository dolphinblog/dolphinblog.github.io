数据类型

Number

JavaScript不区分整数和浮点数，统一用Number表示，这些都是合法的Number类型：

JS支持16进制，十六进制用0x前缀和0-9，a-f表示，例如：0xff00，0xa5b4c3d2等，他们和相应十进制表示的数值完全一样。

Number类型可直接做四则运算。

字符串

字符串是以单引号'或双引号"括起来的任意文本。

布尔值

两个值：true和false

  1. 逻辑运算符
    参考C语言，有&&，||，!等

  2. 比较运算符
    参考C语言，有> > = == !=等

要特别注意==，JS中有两个用于比较是否相等的运算符：

  1. ==比较，会自动转换数据类型再比较
  2. ===比较，不会转换数据类型，如果数据类型不同，直接返回false，如果一致，再进一步比较。

应当总是始终使用===。

null和undefined

null表示一个“空”的值，它和0 ''不同，0是一个数值，''是长度为0的字符串，而null表示”空”

null相当于Python中的None

在JS中还有一个undefined，表示未定义，区分null和undefined二者的意义不大，大多数情况下应该使用null，undefined仅仅在判断函数参数是否传递的时候有用。

strict模式

不用var关键字直接声明变量，比如i = 200，那么i默认就是全局变量。

在同一个页面的不同JS文件中，如果都不用var申明，并恰好都使用了变量i，将造成变量i相互影响，从而难以调试错误结果。

为了修复这一问题，ECMA在JS后续规范中推出strict模式，在strict模式运行下的JS代码，必须使用var申明变量，否则将导致运行错误。
